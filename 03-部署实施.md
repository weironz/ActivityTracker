# Kafka 网站活动追踪系统

## 技术选型
技术选型：

前端：HTML + JS（埋点追踪）
后端：Python + Flask（收集服务 + 推荐接口）
消息队列：Kafka
推荐逻辑：基于用户点击频次的简单推荐

## 架构1

```
浏览器（index.html）
    │  点击/浏览/购买 → POST /track
    ↓
Flask 后端（app.py）           ← Kafka Producer
    │  写入对应 Topic
    ↓
Kafka Broker
    ├── Topic: click
    ├── Topic: page-view
    └── Topic: purchase
    ↓
消费者（consumer.py）          ← 推荐引擎
    │  实时统计行为
    ↓
GET /recommendations → 前端展示推荐商品
```

## 快速启动

### 1. 创建 Kafka Topics

```bash
cd ~/kafka_2.13-4.2.0

bin/kafka-topics.sh --create --topic click --bootstrap-server localhost:9092 --partitions 3 --replication-factor 1
bin/kafka-topics.sh --create --topic page-view --bootstrap-server localhost:9092 --partitions 3 --replication-factor 1
bin/kafka-topics.sh --create --topic purchase --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1
bin/kafka-topics.sh --create --topic user-action --bootstrap-server localhost:9092 --partitions 1 --replication-factor 1
```

### 2. 安装 Python 依赖

```bash
pip install uv --break-system-packages  # 先装 uv
cd backend
uv venv && source .venv/bin/activate
uv pip install -r requirements.txt
```

### 3. 启动后端（终端 1）

```bash
python app.py
```

### 4. 启动消费者（终端 2）

```bash
python consumer.py
```

### 5. 打开前端

```
apt install -y nginx
rm /etc/nginx/sites-enabled/default
chmod o+rx /data /data/codes /data/codes/ActivityTracker /data/codes/ActivityTracker/frontend
```

## 系统效果

页面上点击商品 → 埋点发送到 Flask → 写入 Kafka
底部实时显示 Kafka Event Stream
点击后右侧自动弹出推荐面板
consumer.py 终端实时打印消费日志和推荐结果
